Alert

An alert displays a short, important message in a way that attracts the user's attention without interrupting the user's task.

The alert offers four severity levels that set a distinctive colors. The variant can be specifed as error warning info success classes.

An alert can have an action, such as a close or undo button. It is rendered after the message, at the end of the alert.

Badge

Badge generates a small badge to the top-right of its child(ren).

The default color of badge is red but the color can be changed to green by green-badge class.

Badge can be applied on Text, Icon, Button and Avatar. It defaults to a small rounder dot without any border if no content is specified.

Button

Buttons allow users to take actions, and make choices, with a single click/tap.

Here are few variants of button- Primary, Outlined, Icon, Floating, Link.

These can be achived by btn-primary btn-outlined btn-icon btn-floating btn-link classes respectively.

Vetical Cards and Overlay

Cards contain content and actions about a single subject. They display content and actions on a single topic.

Cards can be Vertical, Horizontal, With or without Images and Overlay.

They can also come in with a badge and dismiss, with actions buttons

card-shadow class can be used to render a card with a shadow. Default cards are set to no shadow and a border
